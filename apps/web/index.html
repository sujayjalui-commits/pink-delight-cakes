Customization rules

Freeze the order rules before backend

  • Keep v1 inquiry-first: customers request cakes, you confirm the final quote manually.
  • Support fixed products and custom styling, but collect design details as a request instead of promising instant checkout.
  • Allow message topper text, candles, and theme color choices as add-ons or notes.
  • Store delivery area, delivery fee, and pickup rules in editable business settings instead of hardcoding them.

V1 backend scope

Build the backend around products, inquiries, admin controls, and business settings.

This keeps the first version lean and avoids building cart, login, payment, and inventory systems before your bakery actually needs them.

Products endpoint

What the product API should expose

Catalog fields Name, slug, category, hero image, description
Pricing fields Starting price, sizes, flavor choices, add-ons
Operational fields Lead time, availability, delivery eligibility
Display fields Badge, featured flag, short marketing summary
Order request endpoint

What the inquiry payload should save

Customer request Product, flavor, servings, event date, delivery choice
Customization Add-ons, topper text, design notes, inspiration link
Status New, quoted, confirmed, scheduled, completed
Channel Website, WhatsApp, Instagram, manual admin entry
Admin needs

Minimum controls for the bakery owner

  • Update product prices, lead times, and availability without touching frontend code.
  • Add or remove flavors, size options, and featured products from an admin view.
  • Review incoming order requests and move them through simple statuses.
  • Edit contact info, delivery area, and delivery fee from business settings.
What to defer

Skip these in v1 unless demand proves you need them

Direct checkout Keep manual quote confirmation for custom orders.
Customer login Not needed for a small, conversation-led bakery flow.
Payment gateway Add later when the order flow becomes more self-serve.
Inventory logic Use simple availability and lead time flags first.
